# 指標（Metrics）の説明

## 1. brightness_mean（平均輝度）
**何を測っているか**
- 画像全体が「どれくらい明るいか」の平均

**計算方法**
- グレースケールの画素値（0〜255）の平均（mean）
- 実装: `np.mean(gray)`

**解釈**
- 高い → 全体が明るい（白っぽい）
- 低い → 全体が暗い（黒っぽい）

**落とし穴**
- 白飛びして情報が潰れていても値は高くなる

---

## 2. contrast_std（コントラスト）
**何を測っているか**
- 明暗の「メリハリ」がどれくらいあるか

**計算方法**
- グレースケールの画素値の標準偏差（std）
- 実装: `np.std(gray)`

**解釈**
- 高い → 明暗差が大きい（メリハリあり）
- 低い → のっぺりしている

**落とし穴**
- ノイズが多いと標準偏差が上がる（"良い"とは限らない）

---

## 3. sharpness_lap_var（シャープネス）
**何を測っているか**
- 輪郭（エッジ）がどれくらいハッキリしているか

**計算方法**
- Laplacianフィルタをかけた結果の分散（variance）
- 実装: `cv2.Laplacian(gray, cv2.CV_64F).var()`

**解釈**
- 高い → ピントが合っている傾向
- 低い → ピンボケ・ブレの傾向

**実測例（sample.jpg）**
- sigma_2: sharpness = 3.63
- sigma_3: sharpness = 2.16
- → ぼかしが強くなるほど値が下がることが確認できる

**落とし穴**
- 平坦な被写体は値が低くなりがち（ぼけ判定に不利）
- ノイズでエッジが増え、値が高く見えることがある
